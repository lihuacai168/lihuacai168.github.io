编辑uWSGI配置文件

配置Django

Django设置多配置文件(生产和开发)
打开Settings目录下的wsgi.py文件

调试uWSGI

配置Nginx

创建uWSGI配置文件

  • 首先进入Nginx的include目录/etc/nginx/conf.d,这个目录下的.conf文件都会被Nginx自动导入

  • 创建uWSGI的配置文件uwsgi.conf,命名是随意的,只要后缀是.conf就行

测试Nginx

  • 启动nginx systemctl start nginx

  • 测试Nginx配置文件 nginx -t

  • 重新加载,使Nginx配置文件生效 nginx -s reload

Supervisor管理uWSGI进程

Supervisor守护进程虚拟环境部署Django

创建Supervisor中的uWSGI配置

  • 创建uWSGI启动的配置文件uwsgi.conf
    command中需要指定uwsgi的绝对路径
    Supervisor配置的注释是;符号

通过Supervisor启动uWSGI

  • Supervisor启动supervisord -c /etc/supervisord.conf

  • 重启所有管理的进程supervisorctl reload

  • 单独启动uWSGI进程supervisorctl start uwsgi

  • 停止uWSGI进程supervisorctl stop uwsgi

uWSGI日志抛出异常Permission denied

修改Nginx启动用户

异常原因分析

  • 修改Nginx启动用户前,默认Nginx用户启动,因为权限问题,无法访问uWSGI的sock文件
  • 查看uWSGI 启动用户,是faster
  • 修改Nginx启动用户为faster(和启动uWSGI的用户一样),就阔以正常访问啦~
